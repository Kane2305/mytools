系統資訊

啟用雲端同步後,每次儲存時資料會上傳到雲端資料庫(Google Firebase / Firestore),同時在此裝置保留一份本地副本。只有持有你同步碼的人能存取雲端上的這份資料,請妥善保管同步碼。

未設定同步碼時,資料只存在這台裝置的瀏覽器裡,清除瀏覽器快取/資料會一併清除所有紀錄。

資料包含:持股記錄、快照歷史、定期定額、衝動日誌、止盈/停損關卡、集中度設定等。建議定期用「匯出備份」留存一份檔案。

連接自己的雲端(進階)

預設情況下,這個工具把資料存在開發者的雲端空間。如果你想要完全獨立、資料存在自己的 Google 帳號底下,照以下步驟做一次(免費,約 15 分鐘):

  1. 前往 firebase.google.com,用 Google 帳號登入,點「Go to console」
  2. 「新增專案」→ 取名 → 關閉 Google Analytics → 建立
  3. 左側找「Firestore」→「建立資料庫」→ Standard → 位置選 asia-east1 → 先選「測試模式」→ 建立
  4. 到 Firestore 上方「規則(Rules)」分頁,整段換成下面這段後按「發布」:
    rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /trackers/{syncId} {
      allow read, write: if true;
    }
    match /{document=**} {
      allow read, write: if false;
    }
  }
}
  5. 回到專案總覽,點 Web 圖示(</>)註冊應用程式,畫面會給你一段 firebaseConfig 設定
  6. 把那整段設定(含 apiKey、projectId 等)複製,貼到下面的框裡,按「套用」

套用後頁面會重新載入,之後所有資料都會存到你自己的雲端。原本雲端上的舊資料不會自動搬過去——需要的話,先用「匯出備份」下載,切換後再「匯入備份」。
